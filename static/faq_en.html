What is the KindleEar?

KindleEar is a free webapp that running in Google Appengine, it aggregate feeds and deliver to your Kindle periodically.

Features:

  • Custom rss
  • Supports fulltext rss and summary rss
  • Multi-account management, which supports many kindles
  • The magazine format MOBI with images
  • Automatic daily scheduled deliver
  • Powerful email service
Benefits:
  • All free, not only this application is free, but GAE provides free hosting space, you do not need to pay a penny.
  • Generate fine magazine format books. Sample as follows (the left one in kindle3, the right two in kindle touch):

This application requires its own server?

This application hosted on Google App Engine server, do not need your own server.

How to deploy my own delivery server?

Following formal steps:

  1. In Github page to download the latest version KindleEar in the lower right corner of the page there is a button "Download ZIP", click to download a ZIP file that contains all the source code, and then extract it to your favorite directory, such as C:\KindleEar (below in this example).
  2. You need a GAE account, please visit Google AppEngine page. Use your gmail account to login, click on "Create Application", in the account activation SMS page, select Others, enter your cellphone number with country code (+nn), click Send, you can receive activation information on the phone in a while, after activate click Create Application again, in the Application Identifier field, enter a name of your choice applications (eg: KindleEar123). The remaining options can keep the default, click on the Create Application to create a GAE application.
  3. Install Python 2.7.x, on this page, select your platform corresponding to the 2.7.x version to download, and then simply double-click the installation. Note Do not install a Python 3.x version, GAE does not support now.
  4. Install GAE SDK, which is used to upload GAE program. in section on this page "Google App Engine SDK for Python", choose your platform, click to download the corresponding version, then install, all the way next without words.
  5. Open Resouce Explorer of your platform, enter the previously downloaded KindleEar directory (eg C:\KindleEar), use a text editor to open app.yaml and module-worker.yaml file, modify the word 'kindleear' in the first line to your GAE application name (eg kindleear123), save and exit.
  6. Modify variables 'SRC_EMAIL' and 'DOMAIN' and 'TIMEZONE' in config.py of same directory of app.yaml. you can modify other variables if you want and understand what it mean, or default value recommented.
  7. Start uploade KindleEar now. you can use Google App Engine Launcher gui or command line. because Launcher is not stable, here is what you use in windows command line.
  8. Followed by clicking: Start - Run - type CMD, and open a command line window
  9. Change current directory into App Engine SDK installation directory. example: cd "C:\Program Files\Google\google_appengine"
  10. Enter 'upload' command: c:\python27\python.exe appcfg.py update C:\KindleEar\app.yaml c:\KindleEar\module-worker.yaml
  11. Enter the gmail account and password (password does not display in screen).
  12. After uploaded successfully use your browser to open domain: http://kindleear123.appspot.com (example only, kindleear123 should be changed to the name of your gae application), if they can open properly, then goto last step.
  13. Add your gmail account to your kindle email whitelist. you should know how to do it before.
  14. Then what? Enjoy your own delivery service now!

Following steps much easy(recommented).

  1. Download KindleEar and uncompress it, get a GAE account, create a application.

Why am I not receive delivery?

For many reasons, the following are few possibilities:

  1. Didn't enable "Enable deliver custom rss" switch, this switch local in bottom of setting page.
  2. Didn't add your gmail address to Amazon 'Approved Personal Document E-mail List'.
  3. The content of rss link isn't a valid XML format.

What is fulltext rss?

Fulltext rss is which contains all content of articles in rss, on the other hand, summary rss contains only brief description of articles. you can confirm it by opening the link of rss.

Can fulltext rss be processed as summary rss? and vice versa?

Fulltext rss can be processed as summary rss, in this case, program ignore the content in rss xml file and fetch fulltext in original link, this would spend much time, resulting in the number of supported rss decreased. If summary rss processed as fulltext rss, articles pushed to your kindle would have brief description only.

How to deal with subscription weekly?

Some rss update weekly, for this type of rss, you can write a py file to directory 'books' with variable 'deliver_days' setted to a Day like 'Friday'.

Can KindleEar push news to Kindle more than one time per day?

The answer is yes, follow steps to realize:

  1. add a python file to directory 'books'.
  2. set property 'deliver_times' to a list which elements are hours you chose.
    for example : deliver_times=[6,14,22] #6:00,14:00,22:00 push three times per day
  3. optional, set property 'oldest_article' to number of interval seconds between two deliveries.
    for example : oldest_article=28800 #8*60*60

  • coverfile = 'books/xxxx.jpg'

How to deal with when occur SSL error in upload procedure?

Some fiends in China would encounter urllib2.URLError: <urlopen error [Errno 8] _ssl.c:504: EOF occurred in violation of protocol>, in this case, change the hosts file of OS, add a line "74.125.229.174 appengine.google.com"

Some articles garbled?

For custom rss and built-books without encoding specified, program detects encoding automatically by using http header, html meta and chardet module of python, the result is not 100% accurate, and if encoding detected is wrong, the article may be garbled. Because this situation is very rare, KindleEar only supply a not very elegant solution like geeker: manually change chardet-detect encoding saved in database. steps follows:

  1. login appid.appspot.com by using account 'admin'(assuming your application name is appid).
  2. input url 'https://appid.appspot.com/dbviewer' in your brower address bar. check encoding of websites in section UrlEncoding. if encoding is wrong for some site, record its ID of database item.
  3. enter a long string in addressbar of brower, 'https://appid.appspot.com/dbviewer?action=modurlenc&id=dbid&feedenc=fenc&pageenc=penc'
      before press enter, modify some vaiable follows:
    • dbid: the ID of database item.
    • fenc: new correct FeedEncoding.
    • penc: new correct PageEncoding.
  4. A method alternate is that change them in 'Datastore viewer' in GAE backend.
  5. You can obtain the correct encoding by viewing source code of a webpage.
  6. If you cannot find the website in page 'dbviewer', it means KindleEar never detect the encoding of any page of this website, http header and html meta are used only. In this case, you can set the variable 'ALWAYS_CHAR_DETECT' in 'config.py' to True.
  7. If article still be garbled after 'ALWAYS_CHAR_DETECT' setted to True, signify encoding detected by chardet is wrong and unmatch of http header and html meta. a final attempt is set 'TRUST_ENCODING_IN_HEADER_OR_META' to True.
  8. After all, if you tested all solutions hereinabove and the problem not solved, please add the rss to a python file to folder 'books' with encoding specified.

How can I do if forgot password?

If you have forgotten the password for non-administrator account, you can notify the administrator to change your password. If an administrator password forgot, you can login appengine.google.com, on the left select "Datastore Admin", delete all KeUser data, all account information will be deleted, and then re-use admin/admin login to your application . Did not find "Datastore Admin"? You can active it in the "Application Settings".
If you know python, you would find other solution.

There are some logs likes 'fetch rss failed(450)' in gae backend, what them mean?

The number in parentheses is a error code, besides standard error code, hereafter shows errorcodes defined by KindleEar.

  • 504:timeout.
  • 509:file too large, GAE has a limit of 4MB for each file/page.
  • 452:website don't support visit by https protocol.
  • 453:websit need to authenticate.
  • 450:download failed.
  • 451:other exception.

Why I cannot lookup words in dictionary when 'Merge books into one' chosen? 

With option 'Merge books in one' checked, KindleEar set the metadata of book as Custom Rss, you have to set the language attribute in section 'Custom Rss Setting'.

How to use mail service xxx@appid.appspotmail.com?

KindleEar supports mail service which address likes xxx@appid.appspotmail.com, xxx is any ascii string, appid is id of your application.

  • KindleEar would transfer content of email receviced to your kindle. if content of email contains hyperlinks only (a link per line), would fetch url(s) to generate a ebook and then push to kindle.
  • Before you can enjoy the feature, please add whitelists in menu 'Advance'|'White List'. A asterisk '*' means allow all sender, or add some addresses like 'xx@xx.xx'/'@xx.xx'.
  • Send mail to username__xxx@appid.appspotmail.com would push ebook to kindle of username if account username exist.
  • Send mail with hyperlinks of ebook to book@appid.appspotmail.com or username__book@appid.appspotmail.com would trigger a task to download ebook and deliver to kindle directly.
    The list of postfix allowed of ebook refers to Mail Python API Overview
  • address trigger@appid.appspotmail.com or username__trigger@appid.appspotmail.com is a trigger, would start a deliver task, which effect when subject is empty or is 'all' equal that you click button 'deliver now' in Menu 'Setting', that place book's title in subject would push book you chose, multi books separated by comma.

Some websites need subscription(login) to view article?

If you need to crawl a website that needs subscription, you have to add a python file to folder 'books', refer to samples books when you don't know how to write a python file.

  1. needs_subscription : Set to True
  2. login_url : Set to URL for login page
  3. form_4_login : Optional, if not set, KindleEar will guess which form in page is used to login, set to a integer indicate the sequence number (where 0 is the first) of the form in page, set to a string indicate the name of form, start with '#' indicate id of form, start with '.' indicate class of form, string only indicate any of name/id/class of form.
  4. Login to KindleEar, In 'Feeds' page, a new button 'Login info' will appear on right side the book which needs_subscription is True in section 'Subscripted'. click it and then provide the account name and password.

Note1: no support site that need run javascript to login or need to input a captcha.

Note2: for some site special, may be you have to rewirte function 'login' in subclass of 'BaseFeedBook'.

Note3: the password will be saved in GAE server with encrypted, key for encrypt is a random string for each account of KindleEar, but key is stored in GAE too, so it's no absolute safe, please take the consequences if the database of KindleEar is stolen.
